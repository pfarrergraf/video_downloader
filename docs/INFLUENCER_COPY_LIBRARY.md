## 1. Nutzungsanleitung / How to Use This Library

### DE

**Platzhalter-Konvention.** Alle Vorlagen verwenden doppelte geschweifte Klammern.
Vor Veröffentlichung jeden Platzhalter ersetzen — nie unausgefüllt posten:

| Platzhalter | Bedeutung |
|---|---|
| `{{creator_name}}` | Anzeigename der Creatorin / des Creators |
| `{{creator_handle}}` | Social-Handle inkl. @ |
| `{{affiliate_link}}` | Persönlicher Affiliate-Link |
| `{{affiliate_code}}` | Persönlicher Partnercode (hat bei der Zuordnung Vorrang) |
| `{{discount_text}}` | Rabatt-Hinweis — **bleibt leer, wenn kein Rabatt existiert** |
| `{{campaign_name}}` | Interner Kampagnenname (Tracking/Absprache) |
| `{{language}}` | Zielsprache des Posts (de/en) |

**Regeln:**

1. **`{{discount_text}}` weglassen, wenn leer.** In v1 des Partnerprogramms gibt es
   **keinen Kundenrabatt**. Den Platzhalter (samt umgebendem Satz) ersatzlos streichen.
   Niemals einen Rabatt, Deal oder Gutschein erfinden.
2. **Kennzeichnung immer an den Anfang.** Die Werbekennzeichnung (Abschnitt 10) steht
   als Erstes — vor dem eigentlichen Text, sichtbar ohne „Mehr anzeigen“ / vor dem Fold,
   nie in einer Hashtag-Wolke versteckt.
3. **Pflichtbaustein Rechte** in jede Veröffentlichung übernehmen (siehe oben).
4. **Nur Aussagen aus der Faktenbasis** verwenden. Nichts hinzufügen: keine Nutzerzahlen,
   Bewertungen, Rabatte, Einnahmeversprechen, keine App-Store-Verfügbarkeit.
5. **Partnercode:** `{{affiliate_code}}` hat bei der Kaufzuordnung Vorrang — Code sichtbar
   nennen, nicht nur den Link.
6. Vorlagen dürfen sprachlich an den eigenen Stil angepasst werden, solange Fakten,
   Kennzeichnung und Pflichtbaustein unverändert korrekt bleiben.

### EN

**Placeholder convention.** All templates use double curly braces. Replace every
placeholder before publishing — never post them unfilled:

| Placeholder | Meaning |
|---|---|
| `{{creator_name}}` | Creator display name |
| `{{creator_handle}}` | Social handle incl. @ |
| `{{affiliate_link}}` | Personal affiliate link |
| `{{affiliate_code}}` | Personal partner code (takes precedence for attribution) |
| `{{discount_text}}` | Discount note — **stays empty if no discount exists** |
| `{{campaign_name}}` | Internal campaign name (tracking/coordination) |
| `{{language}}` | Target language of the post (de/en) |

**Rules:**

1. **Omit `{{discount_text}}` when empty.** Affiliate program v1 offers **no customer
   discount**. Delete the placeholder (and its surrounding sentence) entirely. Never
   invent a discount, deal or coupon.
2. **Disclosure always comes first.** The ad disclosure (section 10) goes at the very
   top — before the body copy, visible without tapping "more" / above the fold, never
   buried in a hashtag cloud.
3. Include the **mandatory rights block** in every publication (see above).
4. **Only use claims from the fact sheet.** Do not add anything: no user counts,
   ratings, discounts, income promises, no app store availability.
5. **Partner code:** `{{affiliate_code}}` takes precedence for purchase attribution —
   state the code visibly, not just the link.
6. Templates may be adapted to your own voice as long as facts, disclosure and the
   rights block remain accurate and intact.
